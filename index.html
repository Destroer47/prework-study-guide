HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Link to a style sheet with "<"link rel="stylesheet" href="./path/to/file.css" /> (Don't include quotes around 1st carrot)
  • You can use classes to change the look/ properties of many different or specific parts of your html file with the style sheet like h1, h2, li, p, and many more
  • In the style sheet classes are added with a "." and id's are added with a "#" followed with the name of the class/ id

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git switch -c branch-name: is how you can save your work and copy it over to a new branch if you fucked up and started writing a bunch of stuff in the wrong branch and before you push or commit to that branch, you can switch to a different branch where you want the stuff to go
  • git add -A: is to commit all files in the working branch
  • git commit -m "Message about changes": is how you commit the changes you made to the working branch/ repo and also describes what changes you made with the message
  • git pull origin main: is the command you use before pushing your changes to the working branch to sync your local branch with the remote branch in github (Doesn't apply when you are working alone)
  • git push origin branch-name: is how you push all your changes to the branch and have the changes show up in GitHub
