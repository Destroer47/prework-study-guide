HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The meta elements stores important meta data for the webpage
  • Using section is good to differentiate different sections of the webpage
  • You can use ordered lists (ol) or unordered lists (ul) to list out items on a webpage
  • You can use img src to link images and also put at alt in the command so that if the image didn't load there would be text to replace the image
  • If needed headers and footers can be great to add info to the top and bottom of the webpage

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Link to a style sheet with "<"link rel="stylesheet" href="./path/to/file.css" /> (Don't include quotes around 1st carrot)
  • You can use classes to change the look/ properties of many different or specific parts of your html file with the style sheet like h1, h2, li, p, and many more
  • In the style sheet classes are added with a "." and id's are added with a "#" followed with the name of the class/ id

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git switch -c branch-name: is how you can save your work and copy it over to a new branch if you fucked up and started writing a bunch of stuff in the wrong branch and before you push or commit to that branch, you can switch to a different branch where you want the stuff to go
  • git add -A: is to commit all files in the working branch
  • git commit -m "Message about changes": is how you commit the changes you made to the working branch/ repo and also describes what changes you made with the message
  • git pull origin main: is the command you use before pushing your changes to the working branch to sync your local branch with the remote branch in github (Doesn't apply when you are working alone)
  • git push origin branch-name: is how you push all your changes to the branch and have the changes show up in GitHub

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • console.log runs an enviornment and executes the JavaScript code which then the output is logged to the console on the webpage
  • You can define variables within an array
  • You can use functions to call upon them whenever needed or at the end of the code
  • You can use for statements to log an array of one of your variables, and define the starting point and increments of what you are going to be selecting from the array
